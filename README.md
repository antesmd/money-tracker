# Money Tracker

Money Tracker — это родительский репозиторий для сборки и запуска приложения для учёта личных финансов. Основной код находится в сабмодулях `backend` и `frontend`, а в этом репозитории собраны общая инфраструктура, конфигурация и инструкции для запуска всего проекта целиком.

## Что входит в проект

- Backend на FastAPI, Python 3.13, SQLAlchemy и Alembic, подключаемый как сабмодуль.
- Frontend на React, TypeScript, Vite и Tailwind CSS, подключаемый как сабмодуль.
- PostgreSQL для хранения данных.
- Nginx как единая точка входа для проксирования запросов.

## Возможности приложения

- аутентификация и работа с JWT;
- управление счетами;
- управление транзакциями;
- работа с бюджетами;
- управление категориями;
- дашборд со статистикой и сводкой по финансам.

## Быстрый запуск через Docker Compose

Это рекомендуемый способ поднять весь проект целиком.

Сначала инициализируйте сабмодули:

```bash
git submodule update --init --recursive
```

Затем запустите все сервисы:

```bash
docker compose up --build
```

После запуска приложение будет доступно по `http://localhost:8000`

Схема запуска в `docker-compose.yaml`:

- сначала поднимается база данных PostgreSQL;
- затем выполняются миграции Alembic;
- после этого стартуют backend и frontend;
- Nginx принимает внешний трафик и проксирует запросы внутрь системы.

## Локальный запуск по частям

Если нужно запускать сервисы отдельно, используйте README в подпапках:

- [Backend](backend/README.md)
- [Frontend](frontend/README.md)

### Backend

```bash
cd backend
uvicorn src.server:app --host 0.0.0.0 --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

## Как получить репозиторий целиком

Если вы клонируете проект с нуля, используйте:

```bash
git clone --recurse-submodules <repo-url>
```

Если репозиторий уже клонирован без сабмодулей, выполните:

```bash
git submodule update --init --recursive
```

## Структура репозитория

- `backend/` — сабмодуль серверной части, миграции и логика доменных модулей.
- `frontend/` — сабмодуль клиентского приложения и UI.
- `nginx/` — конфигурация nginx для единой точки входа.
- `docker-compose.yaml` — оркестрация всех сервисов для локального запуска.

## Примечание по окружению

Для корректной работы backend ему нужны переменные окружения для подключения к базе данных и настройки JWT. При запуске через Docker Compose основные сервисы поднимаются автоматически, но для внешнего запуска backend и frontend могут потребоваться дополнительные настройки из соответствующих README.
